## 2. 数据模型（两层）

### 2.1 表结构草案

```sql
-- 彩种目录（改造现有 lottery_catalog）
lottery_catalog (
  code          VARCHAR(32) PRIMARY KEY,   -- 对内稳定 code，创建后不可变
  display_name  VARCHAR(64) NOT NULL,      -- 对外中文展示名
  category_code VARCHAR(16) NOT NULL,      -- 大类：ffc/jisu/lhc/syxw/pk10/k3/ssc/pc28
  play_template VARCHAR(32) NOT NULL,      -- 玩法模板：ssc_std/lhc_std/...
  ball_count    SMALLINT    NOT NULL,      -- 开奖位数，见 §0 ball_count 表
  draw_interval VARCHAR(16),               -- 1m/3m/5m/jisu，NULL=展示「不定期」（§11-7）
  outbound_lottery_code VARCHAR(64),     -- 对外彩种编码（本平台自定，§11-3）
  sort_order    INT         NOT NULL DEFAULT 0,
  on_sale       BOOLEAN     NOT NULL DEFAULT true,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at    TIMESTAMPTZ NOT NULL DEFAULT now()
)
-- 移除字段：detail_alias

-- 玩法模板（6 套）
play_templates (
  code    VARCHAR(32) PRIMARY KEY,   -- ssc_std / lhc_std / ...
  label   VARCHAR(64) NOT NULL,      -- 展示名
  version INT NOT NULL DEFAULT 1
)

-- 第一层：玩法类型
play_types (
  id            BIGSERIAL PRIMARY KEY,
  template_code VARCHAR(32) NOT NULL,  -- FK play_templates.code
  type_id       VARCHAR(32) NOT NULL,  -- dingwei/qian3/zhong3/...（模板内唯一）
  label         VARCHAR(32) NOT NULL,  -- 定位胆/前三/中三/...
  sort_order    INT NOT NULL,
  panel_type    VARCHAR(32),           -- 投注面板类型【待定】
  enabled       BOOLEAN NOT NULL DEFAULT true,
  created_at, updated_at
  -- UNIQUE(template_code, type_id)
)

-- 第二层：子玩法
sub_plays (
  id            BIGSERIAL PRIMARY KEY,
  template_code VARCHAR(32) NOT NULL,
  type_id       VARCHAR(32) NOT NULL,  -- 关联 play_types.type_id（同模板内）
  sub_id        VARCHAR(64) NOT NULL,  -- qian3_zhixuan_fs 等（模板内唯一）
  label         VARCHAR(64) NOT NULL,  -- 前三直选复式
  sort_order    INT NOT NULL,
  bet_mode      VARCHAR(24),           -- 结算分类：fushi/danshi/hezhi/kuadu/zuxuan/...
  segment_rule  JSONB,                 -- 引擎规则（位段、组选模式、第三方赔率键等）
  enabled       BOOLEAN NOT NULL DEFAULT true,
  created_at, updated_at
  -- UNIQUE(template_code, type_id, sub_id)  -- 允许不同大类下同名 sub_id（如「复式」），靠 type_id 区分
)
```

**子玩法展示（§11-11）**：同一大类下 `label` 用短名（如「复式」），大类之间不重复；库表唯一键 **`(template_code, type_id, sub_id)`**，故不同大类可共用短 `sub_id`（如 `erquanzhong/fushi` 与 `sanzhonger/fushi`）。

### 2.2 与现有结构对照

| 现状 | 迁移后 |
|------|--------|
| `lottery_scheme_option_sets.play_types`（扁平 JSON 4 项） | `play_types` 表（SSC **15** 项） |
| `lottery_scheme_option_sets.sub_plays`（扁平 JSON 3 项） | `sub_plays` 表（全平台 **340** 项） |
| `lottery_scheme_option_sets.run_types`（定码轮换等） | **保留 JSON 存储**；枚举 **固定 7 种**（§2.4），与玩法树正交 |
| `playMethod` 拼接「定位胆万位」 | `type_id + sub_id` 主导；`playMethod` 仅展示/审计 |

### 2.3 `segment_rule` 字段约定（草案）

| 字段 | 必填 | 说明 |
|------|------|------|
| `outboundPlayCode` | 是（P1 seed） | **本平台定义**的对外玩法编码；与 `sub_id` 可相同或独立，**一经发布慎改** |
| `numberPoolMin` / `numberPoolMax` | 按模板 | 如 11选5：`1` / `11` |
| `pickCount` | 按模板 | 如 11选5：从 11 个号里选 5 个 |
| （其它引擎键） | 否 | 位段、组选模式等，P2 结算实现时补 |

> `outbound_play_code`：**`{template_code}:{type_id}:{sub_id}`**（C3）。彩种级见 `lottery_catalog.outbound_lottery_code`（C6、C20）。

### 2.4 方案运行类型（固定，与玩法树正交）

新方案可选 **运行类型** 固定为以下 **7 种**（§11-13）：

| # | 名称 |
|---|------|
| 1 | 定码轮换 |
| 2 | 高级定码轮换 |
| 3 | 高级开某投某 |
| 4 | 冷热温出号 |
| 5 | 随机出号 |
| 6 | 内置计画 |

## 13. 术语说明（回应 Q5「从哪里复制、复制到哪里」）

### 13.1 「粘贴」——不是复制方案

**不涉及**「旧彩种复制到新彩种」（该流程已废止）。仅指 **选号文本** 的贴入：

```
┌─────────────────┐         ┌──────────────────────────────┐
│ 用户剪贴板       │  粘贴    │ 本系统选号输入框              │
│ （Excel/备忘录/   │ ──────► │ groupContent textarea        │
│  其它 App 复制的 │         │ 例：「1,2,3」或「12,34,56」   │
│  号码文本）      │         └──────────────────────────────┘
└─────────────────┘                      │
                                         ▼
                              提交投注 / 保存方案组
```

| 页面 | 从哪来 | 贴到哪 | 贴的是什么 |
|------|--------|--------|------------|
| 高级方案编辑 | 剪贴板或手打 | 该组 **textarea**（**无**专用粘贴按钮，C16） | 一组选号文本 |
| 游戏详情·手动下注 | 剪贴板或手打 | 投注区 **textarea** | 当期选号 |
| 快三·手动输入 | 剪贴板或手打 | 对应输入框 | 骰子/号码文本 |

**与以下无关**：复制整份方案、旧彩种迁新彩种、跟单 fork。

### 13.2 「复制」——已废止的含义

| 曾指什么 | 现产品决策 |
|----------|------------|
| 复制 **旧彩种方案** 为新彩种方案并做玩法映射 | **取消**；旧数据 **purge 删除**（C13） |
| 同彩种下 fork 一份 **新云端方案**（现网能力） | **保留**；仅作用于 **47 新彩种**（与迁移无关） |
