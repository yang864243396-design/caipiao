# 方案运行类型（runTypeId）差异化实现规划

> 状态：**v8 实施中**（产品五轮确认完毕，见 §9 决议记录；2026-06-10 开始实施：后端 P0-P4 + S2 已完成（迁移 00082/00083、引擎策略 worker_pick.go、收藏接口、倍投编译，单测全绿），前端四工作流进行中）
> 依据：产品 docx《运行类型和方案内容的关联》+ 2026-06-10 五轮产品确认（Q1-Q9、P1-P6、R1-R4、S1-S2、T1-T2）；UI 改造遵循 `client/DESIGN.md`（数字精算主义）
> 关联代码：`backend/internal/schemes/worker.go`、`worker_config.go`、`worker_lookback.go`、`update_definition.go`、client `CustomSchemeNewView.vue` / `AdvancedSchemeEditView.vue` / `AdvancedSchemeRoundsView.vue` / `BetMultiplierSettingsView.vue`

## 0. 总原则

**一、倍投体系与运行类型出号体系相互独立、互不影响【P1】：**

- 倍投体系：决定**每期下多少倍**；运行类型体系：按 `runTypeId` 决定**每期下什么号码**。
- 引擎并行推进两个状态机：倍率轮次游标按命中/未中走 `rounds`；出号游标（组/局/池/投向）按各类型规则走。
- **例外【R1】：回看止盈止损触发重置时，两套游标一并复位**——倍率回第 1 轮（现状）+ 出号游标复位（局数回第 1 局、号码池刷新、投向回正投、组游标回第 1 组）。
  - 复位范围与现状轮次重置对称：**个别判断**复位触发实例；**整体判断**复位该会员同运行模式下**全部运行中实例**的双游标（`applyLookbackResets` 现状即重置全部实例的倍率轮次）。

**二、倍投设定打通【S2，现状缺口修复】**：现状引擎仅消费「轮次设置」`rounds`，「倍投设定」4 Tab（小白/一键/简单/高级，存 `betMultiplier`）**存而不用**。本次纳入范围：**4 Tab 配置统一编译为 `rounds` 供引擎消费**。默认编译规则：倍数序列按「未中进下一级（afterMiss=下一行）、命中回第一级（afterHit=第 1 行）」生成；轮次设置页直接编辑 rounds 的能力保留，两入口最终落同一份 `rounds`（后保存者覆盖，编辑页需提示）。

**三、适用范围：运行类型分发仅对 `kind=custom`（自创方案）生效**；`contrary`（反买）、`follow`（跟单）方案维持现有专属逻辑，不参与 runTypeId 分发。

**四、整池命中口径**：号码池/多注内容当期**任一注命中即视为「中」**（用于中后换/挂后换/中奖轮换/倍率跳转判定；与现有 eval 行为一致）。

## 1. 背景与现状

- `runTypeId` 仅写入 `scheme_definitions.config`，云端 worker 不读取，所有方案统一按「组轮换 + rounds 倍投 + 回看重置」一套引擎运行。
- `scheme_instances.round_index` 同时充当轮次游标和组游标，差异化后须解耦。
- 旧枚举 `batch_fixed`（批量定码）、`dynamic_chase`（动态追号）、`plan_follow`（计划跟投）**废弃**，存量映射见 §8。
- `scheme_share_snapshots` 表无源方案关联字段（纯静态配置快照）——内置计画据此定为「配置复制」路线【S1】。
- 现有编辑页能力可复用：方案内容提示随玩法变化（`groupContentPlaceholder` + playConfig，已支持龙虎「龙/虎/和」）、注数计算（`validateGroupContent` / `countBetUnits`）。

## 2. 运行类型定版（7 种）与玩法关联矩阵

> 玩法关联矩阵以**代码常量**维护（全局统一，改动走发版）【Q1】。运行类型创建后**不可修改**，选错删除重建【P5，维持后端 PATCH 黑名单现状】。

| value（建议） | label | 支持玩法 | 出号来源 |
|---|---|---|---|
| `fixed_rotate` | 定码轮换 | 所有玩法 | 多组号码每期轮换 |
| `adv_fixed_rotate` | 高级定码轮换 | 所有玩法 | 局数列表驱动（见 §3.2） |
| `adv_trigger_bet` | 高级开某投某 | **仅定胆位（dingwei）与龙虎（longhu）** | 上期开奖触发「开 X → 投 Y」映射 |
| `hot_cold_warm` | 冷热温出号 | **所有玩法，除龙虎** | 频次分档选号组成号码池 |
| `random_draw` | 随机出号 | 所有玩法 | 按位随机生成号码池 |
| `builtin_plan` | 内置计画 | **不选玩法**（随收藏方案带出） | 收藏方案快照配置复制运行【S1】 |
| `fixed_number` | 固定号码 | 所有玩法 | 单注固定号码每期复投 |

- 矩阵落地：新建方案页选定运行类型后过滤玩法两列；`builtin_plan` 隐藏玩法两列且**后端创建校验放宽**（`lotteryCode` / `playTypeId` / `subPlayId` 均允许为空，待选择收藏方案后由快照带出物化覆盖）；其余类型后端 `CreateScheme` 按矩阵校验。
- **反向联动**：运行类型选项随所选彩种的玩法树可用性禁用——彩种无定胆位/龙虎玩法（如 K3、PC28、六合彩模板）时「高级开某投某」置灰不可选；彩种调整后已选运行类型失效时清空重选。
- **位面板首期覆盖全部玩法模板**（SSC 5 位、PK10、K3、LHC、PC28、11 选 5 等按模板动态适配）【P4】，位数/标签取自玩法树 `segmentLen / segmentLabels`。

## 3. 各类型行为与「方案内容」编辑规格（定版）

> 落点：client `AdvancedSchemeEditView.vue` 方案内容区按 `runTypeId` 切换面板；「倍投设定」「轮次设置」入口对全部类型保持（编译归一见 §0-二）。UI 遵循 `client/DESIGN.md`：Element Plus 组件优先、主色 `#0050cb`、色阶分层不拉实线、容器内边距 ≥1rem。
> 注数与注额：一律复用现有玩法注数规则（`countBetUnits`：定位胆按位求和、直选复式为各位笛卡尔积等），注额 = 2 元 × 注数 × 当期倍数。

### 3.1 定码轮换 `fixed_rotate`
- 方案内容为**数组（多分组）**：可新增/删除分组；输入提示随玩法变化（复用 `groupContentPlaceholder`）；存储沿用 `schemeGroups`。
- 引擎【Q2 = C】：**每期必换下一组**（组游标按期 +1 循环），倍率独立走倍投体系。
- 存量影响【R4 = 接受】：现有自创方案默认即此类型，换组节奏由「随命中跳转」统一改为「每期必换」。
- UI：现状分组卡片即此形态，微调即可。

### 3.2 高级定码轮换 `adv_fixed_rotate`
- **局数列表模型【Q3】**：每行 = `局数`（唯一序号）+ `投注号码`（提示随玩法变化）+ `中后跳转至第几局` + `挂后跳转至第几局`。
- 引擎按「当前局数」取该局**号码**下注；结算后按命中/未中跳局；**跳转目标局不存在 → 回第 1 局**。
- 【P1】局数列表只管**号码**跳转；倍率仍由倍投体系独立决定，两游标并行。
- UI：方案内容区新面板——`el-dialog`/抽屉录入（局数、号码、中后、挂后）+ 列表行展示与删除；**不动现有轮次设置页**。

### 3.3 高级开某投某 `adv_trigger_bet`
- 方案内容为**固定展示的映射列表**：每行 = `启用开关`（默认启用）+ 开出号码 → 正投号码 + 反投号码；**每格均为单个号**【P3】；正/反投号码由用户独立填写【Q4a】；**未启用的行不参与触发**【T1】。
- 触发判据：定胆位玩法 = 上期**所选子玩法对应位**开出的号码；龙虎玩法 = 上期龙虎结果（龙/虎/和）。
- 顶部「全部随机」【R2】：**编辑页的一次性辅助操作**——点击时为映射表的正投/反投格批量随机生成单号填入，属纯前端填表功能；**引擎下注时不涉及随机**，不作为运行参数存储。
- 底部投向模式四选一：一直正投 / 一直反投 / 前正后反 / 前反后正。
  - 【Q4b】「前/后」指**前一局投向**：按上一局正/反决定本局（交替状态机，自正投起始）。
- 【Q4c】上期开出号码无启用的映射行时：按**启用行中第 1 行号码的正投**下注。
- UI：紧凑映射表（el-table 用 `min-width`）；模式 `el-radio-group`；「全部随机」置面板头部右侧。

### 3.4 冷热温出号 `hot_cold_warm`
- 分档【Q5a = A】：统计最近「总期数」内频次，**排序后三等分**：热（最近常开）/ 温（普通）/ 冷（长期不开）；显示几位由玩法决定，**多位玩法（如直选复式）每位独立统计三档、每位独立选号**。
- 选号【Q5b】：用户从三档勾选号码**组成下注号码池**。
- **下注口径【P2 = B】：整池一次下注**（注数按玩法注数规则计算）。
- 「中奖轮换」（可开关）：当期任一注命中后**刷新/轮换池内容**（在所选档位号码内轮换出新池）。
- 底部「总期数」可调，调整后重新统计刷新三档。
- UI：三档色区用主色色阶（热 primary、温 light-5、冷 light-9）；总期数 `el-input-number`；已选号码池单独成行展示并显示注数。

### 3.5 随机出号 `random_draw`
- 生成【Q6】：按玩法显示各位面板；**每个位置可配号码数量（1-10）**，点「生成」随机产出号码**组成号码池**；生成后可手动调整。
- **下注口径【P2 = B】：整池一次下注**（注数按玩法注数规则计算）。
- 换号策略四选一（对整池生效，命中口径=任一注中）：每期换（每期引擎重新随机）/ 不换号 / 中后换 / 挂后换。
- 引擎侧随机结果须在下单事务前确定并落库（`bet_content`），可审计。
- UI：位面板 + 每位数量 `el-input-number`(1-10) + 「生成」主按钮；策略 `el-radio-group`。

### 3.6 内置计画 `builtin_plan`
- 【P6】收藏对象 = **跟单大厅中的方案**（分享快照，含彩种 + 玩法），收藏入口 = **玩法详情页星标**（仅从跟单大厅进入、带 snapshotId 时星标可用并对接后端；其他入口不显示收藏）。
- **运行机制【S1 = B 配置复制】**：用户在方案内容中选择已收藏方案后，**将该快照的配置（彩种、玩法、方案内容、运行参数）复制进本方案，在本会员侧独立运行**——非实时跟单，与目标方案彻底解耦：
  - 目标方案停跑/删除/取消公开均**不影响本方案**【R3 自然满足】；
  - 快照为分享时点的静态配置，源方案后续修改**不同步**（产品已接受）；
  - 引擎无需特殊取号策略：物化后按快照内嵌配置的实际出号规则执行。
- 前置【Q7 = A】：收藏后端化（`member_scheme_favorites` 表 + 收藏/取消/列表接口）。
- **复制后的配置只读**【T2】：保持与收藏计划一致，不允许编辑物化配置；用户想自定义需改用其他运行类型重建。
- UI：收藏方案选择列表（彩种、玩法、近期战绩摘要）；选择后展示已复制的配置概要（只读态）；空态引导「先去跟单大厅收藏方案」。

### 3.7 固定号码 `fixed_number`
- 方案内容**仅需设置一注号码**，每期原样复投；存储复用 `schemeGroups`（单元素）；倍率独立走倍投体系【Q8 = A，P1】。
- UI：单组号码输入卡片，提示随玩法变化。

## 4. 引擎改造设计

`placePeriodBet` 按 `config.runTypeId` 分发**出号策略**（仅 `kind=custom`）；倍率统一走 rounds（含倍投设定编译产物【S2】）：

```go
// 出号策略：返回本期下注内容（号码池/单组）与结算后的出号游标推进
type pickStrategy func(cfg parsedSchemeConfig, st pickState, draw Draw, hist []Draw) (betPick, pickState, error)
```

| runTypeId | 出号策略 | 出号游标推进 | 依赖数据 |
|---|---|---|---|
| fixed_rotate | groupRotate | 每期 +1 循环 | — |
| adv_fixed_rotate | jushuList | 中后/挂后跳局，缺局回第 1 局 | — |
| adv_trigger_bet | triggerMap | 投向状态机（记上一局正/反），缺映射走第 1 行正投 | 上期开奖 |
| hot_cold_warm | freqPool（整池） | 中奖后池内轮换刷新（开关） | 最近 N 期开奖（`lottery_draws` 已有） |
| random_draw | randomPool（整池） | 换号策略（每期/不换/中后/挂后） | 上期命中结果 |
| builtin_plan | 物化为内嵌配置的实际类型执行【S1】 | 随内嵌类型 | 收藏表 + 快照 config |
| fixed_number | singleFixed | 无 | — |

- 未识别 / 缺省 `runTypeId` → 按 `adv_fixed_rotate` 兜底；`kind=contrary/follow` 不走分发，维持现有逻辑。
- **回看重置【R1】**：止盈止损触发时，倍率轮次回第 1 轮（现状）+ 出号游标一并复位（`pick_index=0` / `current_pick` 清空重生成 / `last_direction` 回正投），落点 `worker_lookback.go` 的 `applyLookbackResets`。

实例新增运行时状态：

| 字段 | 用途 |
|---|---|
| `pick_index` | 定码轮换组游标 / 高级定码轮换当前局数 |
| `current_pick` | 冷热温/随机的当前号码池（跨期保号：不换号/中后换/挂后换） |
| `last_direction` | 开某投某上一局投向（正/反） |

> 3.【S1 = 配置复制】内置计画非实时跟单：快照为分享时点静态配置，源方案后续调整不同步；如未来要做「实时跟投」，需快照表加源方案关联并处理同期时序，另行立项。
> 4.【S2】倍投设定 4 Tab 此前存而不用，打通后用户历史保存的 `betMultiplier` 将首次实际生效，上线前需扫库评估受影响方案并在公告中说明。

## 8. 兼容性与迁移

- 存量映射【Q9 = B】：废弃的 `batch_fixed` / `dynamic_chase` / `plan_follow` **全部统一映射到 `adv_fixed_rotate`**。
  - 配置换形：存量 `schemeGroups[i]` 生成局数列表（局 i+1 = 第 i 组号码，中后/挂后默认回第 1 局）；存量 `rounds` 留在倍投体系原样不动【P1】。
- 存量 `fixed_rotate` 方案保留类型不迁移，按新引擎「每期必换组」执行【R4】。
- 存量 `betMultiplier` 配置：迁移时按 §0-二规则编译为 `rounds`（已有 rounds 的以 rounds 为准，不覆盖）【S2】。
- 兜底：未识别值按 `adv_fixed_rotate` 运行，引擎不报错；`kind=contrary/follow` 不受影响。
- 迁移脚本幂等；`pick_index` 回填自 `round_index`。
