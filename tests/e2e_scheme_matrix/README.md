# 自创方案全矩阵 E2E 测试

> **方案基线**：v1.1（已锁定）  
> **自动化**：Playwright 有头双浏览器（用户端 + 第三方 v6hs1）  
> **Python**：建议 3.11+（开发机可用 3.12–3.14）

本文档是测试方案与脚本模块设计的唯一说明；实现须与此对齐，变更需先改文档再改代码。

---

## 1. 目标

全量覆盖用户端「游戏大厅 → 自创方案」在**前端可选**的：

`彩种 × 运行类型 × 玩法类型 × 子玩法`

（高级开某投某再 × 4 种触发模式），完成：

1. 创建方案并与第三方预览**注数**对齐  
2. 云端中心**正式运行**开启  
3. 按真实开奖节奏跑期  
4. 对比本平台与第三方投注/开奖记录  
5. 暂停方案（不删除）  
6. 输出 Excel；支持断点续跑与中断全停  

**首次执行**：先冒烟 **1 个彩种 × 1 个子玩法**（含完整目标期数），通过后再全量。

## 3. 覆盖矩阵

### 3.1 运行类型

| 运行类型 | 是否纳入 |
|----------|----------|
| 定码轮换 | 是 |
| 高级定码轮换 | 是 |
| 高级开某投某 | 是（见 3.2） |
| 冷热温出号 | 是 |
| 随机出号 | 是 |
| 固定号码 | 是 |
| 内置计画 | **否** |

组合来源：自创方案页 UI 枚举——按彩种切换运行类型，采集该运行类型下前端可选的玩法/子玩法（不可选则不测）。

### 3.2 高级开某投某

每个子玩法生成 **4** 条用例：

- 一直正投 `always_pos`
- 一直反投 `always_neg`
- 前正后反 `alt_pos_first`
- 前反后正 `alt_neg_first`

「全部随机」仅作配置页填表辅助，**不单独成案**。

目标期数 = `ceil(该彩种全局目标期数 / 4)`（例如全局 10 → 每模式 3）。

### 3.3 目标期数（`draw_interval`）

数据来源：登录后调本平台彩种相关 API（与页面同源）。

| `draw_interval` | 处理 |
|-----------------|------|
| `1m` / `3m` / `5m`（≤5 分钟，含 5m） | 目标 **10** 期 |
| 明确 >5 分钟 | 目标 **10** 期 |
| `jisu` / 空 / 无法解析 | **跳过**，Excel 记原因 |

必须按真实开奖节奏等待（不加速）。

## 4. 单案固定参数

| 字段 | 值 |
|------|-----|
| 方案名称 | 每次：`{彩种}-{运行类型}-{玩法}-{子玩法}_{yyyyMMddHHmmss}`；高级开某投某再加模式后缀（如 `-一直正投`） |
| 分享状态 | 公开 |
| 方案资金 | 100000 |
| 开始/结束时间 | 空（无限期） |
| 止损 / 止盈 | 100000 / 100000 |
| 倍数系数 | 1 |
| 投注单位 | 界面选项「1元」 |
| 方案模式 | 简单倍投（进入设定页后用**默认细项**，仅选中并保存返回） |
| 运行模式 | 正式运行 |
| 选号 | 输入框 + 按钮都做；可增减注数；号码集合与第三方一致 |
| 注数断言 | **以第三方预览注数为准** |
| 特殊运行类型额外参数 | 有默认则用默认；无默认导致无法出注则跳过并记原因 |

创建阶段第三方：**只选号看预览，不点投注**。跑期由本平台挂机自动向第三方下注。

## 5. 第三方操作路径

1. 登录 https://www.v6hs1.com/  
2. 顶部彩种菜单弹窗 → 点击对应彩种 → 详情页选号并读预览注数  
3. 投注记录：https://www.v6hs1.com/user/betRecord  
4. 当前无图形验证码 / 短信 / 二次验证  

彩种名对齐：对本平台展示名做**归一化模糊匹配**（去空格、数字与中文数字互转等如「一分彩」↔「1分彩」）；匹配失败则该彩种相关用例跳过并记原因。

## 6. 调度与生命周期

| 项 | 规则 |
|----|------|
| 并行 | 整条链路（创建→开启→跑期→对比→暂停）占 1 槽，**最多 3** |
| 失败 | 记录原因，**继续**下一组合 |
| 结束后 | **暂停**方案，**不删除** |
| Ctrl+C / SIGTERM | **立即暂停**本批全部已开启测试方案，并落盘进度 |
| 断点续跑 | 仅跳过 `record_compare = passed` 的组合；其余可重试 |
| 亏损上限 | **不设** |

等待超时建议：

- 单期记录出现：最多等 **2 × 开奖间隔**  
- 整案：最多 **目标期数 × 间隔 × 1.5**  

提前终止（止盈/止损/资金不足等）：状态记「提前终止」，仍对比**已产生**记录。

## 7. 断言与对比

### 7.1 创建时

第三方预览注数 vs 本端注数；以第三方为准，不一致则本行失败。

### 7.2 开启

云端实例进入运行中。

### 7.3 终态逐期（本端 API × 第三方 betRecord）

配对键：**同一彩种 + 同一期号**。

比对字段：

- 期号  
- 投注号码/内容  
- 注数  
- 金额（精度 **0.01**）  
- 中奖状态 / 派奖金额  
- 开奖号码  
- 倍数  

本端投注记录：允许在已登录会话下调用本平台 API；创建/开启/第三方选号与 betRecord 抓取仍用 UI。

## 10. 主流程（单用例）

```text
双方登录
→（续跑：跳过 record_compare=passed）
→ UI 已枚举的组合 + API 读 draw_interval
→ 大厅自创方案：选组合 → 填配置 → 简单倍投默认 → 选号
→ 第三方同彩种同号码选号（不下单）→ 注数断言
→ 保存入云 → 开启（并行≤3）
→ 真实开奖等待至目标期数或提前终止
→ 本端 API 记录 × 第三方 /user/betRecord
→ 暂停方案 → 更新 progress + 追加 Excel → 释放槽位
```

中断：暂停全部已开启实例 → flush → 退出。

## 12. 本地运行

```bash
cd tests/e2e_scheme_matrix
python -m venv .venv
# Windows:
.venv\Scripts\activate
pip install -r requirements.txt
playwright install chromium
copy .env.example .env   # 并填写密码
python main.py --smoke   # 1 组合完整链路：建案+注数+开启+跑期+对账+暂停
python main.py           # 全量（默认并行=1，见 config.MAX_PARALLEL）
python main.py --resume  # 断点续跑（仅跳过 record_compare=passed）
```

常用参数（以 `python main.py -h` 为准）：

- `--smoke`：仅 1 个可用组合（跳过无 draw_interval 的），含完整目标期数等待  
- `--resume`：读取 `progress.json` 跳过已终态通过  
- 默认有头；`--headless` 可关界面  
- 并行度：`config.MAX_PARALLEL`（默认 1，降低建案/页签超时）  
- Ctrl+C：立即暂停全部已跟踪云端实例并落盘
