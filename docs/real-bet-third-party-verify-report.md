## 5. game_id / 彩种映射结论

对 **50 条成功下单** 与 [hash.iyes.dev](https://hash.iyes.dev/) `GET /api/web_bets/` 回读对比：

| 对比项 | 结论 |
|--------|------|
| `game_id` 数值 | 与本地 `lottery_catalog.outbound_lottery_code` **完全一致**（含 tron_ffc_1m=27、hash_ffc_1m=21） |
| `game_name` | 成功项中 **game_id 与名称一一对应正确**；本次实测 tron/hash 1分彩 **未出现** 第三方反馈的名称对调 |
| `rule_id` | 与本地 `outbound_play_code` **完全一致** |
| `bet_content` | 与本地编码 **完全一致** |
| `bet_amount` / `bets_nums` | 与下单参数 **完全一致**（多数 2 元×1 注；PC28 和值 4 元×2 注） |
| `periods` | 部分彩种本地占位期号与第三方差 1 期（封盘边界），**以第三方 periods 为准**，不影响注单字段匹配 |

第三方反馈「波场1分彩 / 哈希1分彩 game_id 弄反」：本次通过 `web_bets` API 回读，**存储层 game_id 未反**；若仅在第三方前台展示异常，需第三方修复 UI，**本平台 outbound 映射无需调整**。
